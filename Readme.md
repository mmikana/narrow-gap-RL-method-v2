# 无人机窄缝穿越强化学习仿真环境项目总结

## 一、项目概述

**项目名称**：narrow-gap-RL-method-v2

**项目定位**：完全自主开发的四旋翼无人机仿真环境，用于训练无人机使用强化学习算法自主穿越狭窄缝隙。

**核心价值**：实现了从无人机动力学仿真、环境建模、碰撞检测到 RL 训练框架的完整闭环，为无人机敏捷飞行控制研究提供了可靠的仿真平台。

## 二、核心模块

### 2.1 无人机动力学模型 (`Env/QuadrotorDynamics.py`)

完整的四旋翼六自由度动力学仿真：

- **状态变量**：位置 (3) + 姿态 (3) + 线速度 (3) + 角速度 (3)
- **动力学建模**：
  - 电机推力计算：$F = C_T \cdot \omega^2$
  - 扭矩计算：$\tau = C_M \cdot \omega^2$
  - 空气阻力：$F_{drag} = -k_d \cdot v \cdot ||v||$
  - 重力与科里奥利力耦合
- **数值积分**：RK4 四阶龙格库塔方法保证仿真精度
- **电机模型**：一阶惯性滞后响应 $ \omega_{curr} = c \cdot \omega_{prev} + (1-c) \cdot \omega_{cmd} $

```python
# 关键参数
mass = 1.2 kg          # 无人机质量
arm_length = 0.33 m    # X 型机臂对角线长度
inertia = diag(0.07, 0.07, 0.14) kg·m²  # 惯性矩阵
C_T = 1.1e-6           # 推力系数
C_M = 1.4e-7           # 扭矩系数
dt = 0.02 s            # 仿真步长
```

### 2.3 碰撞检测系统 (`Env/collision_detector.py`)

基于**分离轴定理 (SAT)** 的精确碰撞检测：

```
┌─────────────────────────────────────┐
│  两阶段检测策略                      │
├─────────────────────────────────────┤
│  1. 快速距离筛选 (simple_distance_check) │
│     → 过滤明显无碰撞情况              │
│  2. 精细 SAT 检测 (check_channel_collision) │
│     → 4 个平面 × 多分离轴             │
└─────────────────────────────────────┘
```

**创新点**：
- 严格区分平面内外侧判断逻辑
- 动态容差补偿处理浮点误差
- 支持倾斜缝隙的精确检测

### 2.4 强化学习环境 (`Env/Env1.py` - `Quad2NGEnv`)

符合 **Gymnasium** 标准接口的 RL 环境：

#### 状态空间 (15 维，归一化)
```
obs = [相对位置 (3), 速度 (3), 姿态 (3), 角速度 (3), 缝隙姿态 (3)]
```

#### 动作空间 (4 维)
```
action = [归一化电机转速] ∈ [-1, 1]⁴
```

#### 复合奖励函数
| 奖励项 | 公式/说明 |
|--------|-----------|
| 位置奖励 | $r_{pos} = \frac{1}{1 + w_{pos} \cdot ||p_e||^2}$ |
| 姿态奖励 | $r_{ori} = \sum \frac{trigger\_p}{1 + w_{ori} \cdot |\phi_e|}$ |
| 速度奖励 | $r_{speed} = -w_{speed} \cdot \frac{|v - v_{ideal}|}{v_{ideal}}$ |
| 电机惩罚 | $r_{motor} = -w_{motor} \cdot \sum|\Delta\omega|$ |
| 碰撞惩罚 | -200 |
| 成功奖励 | +100 |

#### 课程学习系统
**27 个难度等级**渐进式训练：
```
阶段 1-3:  L1 (3.0×缝隙) + R_A/B/C × T_A/B/C
阶段 4-6:  L2 (2.5×缝隙) + R_A/B/C × T_A/B/C
阶段 7-9:  L3 (2.0×缝隙) + R_A/B/C × T_A/B/C
```

### 2.5 SAC 强化学习算法 (`SAC_agent.py`)

完整实现 **Soft Actor-Critic** 算法：

```
┌──────────────────────────────────────────────┐
│              SAC Agent 架构                   │
├──────────────────────────────────────────────┤
│  Actor Network    → 高斯策略输出             │
│  Critic Network 1 → Q 值估计 (双网络抑制高估)  │
│  Critic Network 2 → Q 值估计                 │
│  Value Network    → 状态值函数               │
│  Target Network   → 软更新目标               │
└──────────────────────────────────────────────┘
```

**网络结构**：
- Actor: FC(256) → FC(256) → μ,σ输出
- Critic: FC(256) → FC(256) → Q 值
- Value: FC(256) → FC(256) → V 值

**训练技巧**：
- 经验回放缓冲区 (10,000 条)
- 梯度裁剪 (max_norm=1.0)
- 软更新系数 τ=0.05
- 探索噪声自适应衰减

## 五、关键设计决策

### 5.1 为什么选择 SAC 算法？
- **最大熵框架**：鼓励探索，适合连续控制任务
- **离线策略**：可复用历史数据，样本效率高
- **双 Critic**：有效抑制 Q 值高估问题

### 5.2 为什么设计课程学习？
- 窄缝穿越是**稀疏奖励**任务
- 直接从高难度开始训练难以收敛
- 27 关卡设计覆盖：
  - 缝隙尺寸变化 (3.0× → 2.0×)
  - 旋转角度变化 (20° → 60°)
  - 倾斜角度变化 (10° → 30°)

### 5.3 为什么使用 RK4 积分？
- 欧拉法误差累积快，长期仿真不稳定
- RK4 四阶精度保证动力学仿真可靠性
- 计算开销可接受 (dt=0.02s)
